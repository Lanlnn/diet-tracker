# 管理后台 · UI/UX Pro Max 方案

这是独立于小程序代码的管理后台设计原型，用于与 Product Design 方案做同范围对照。直接在浏览器中打开 `index.html` 即可预览。

## 本版设计依据

- 设计系统：`design-system/diet-tracker-admin-·-ui-ux-pro-max/MASTER.md`
- Dashboard 覆盖规则：`design-system/diet-tracker-admin-·-ui-ux-pro-max/pages/dashboard.md`
- 风格：Data-Dense Dashboard，青绿健康品牌色，浅/深色双主题
- 信息结构：运营总览、用户管理、食品库、内容审核、运动配置、运营规则、系统状态
- 关键体验：全局筛选、KPI 对标、趋势数据表替代、转化漏斗、待审核队列、系统健康状态

## 可交互内容

- 切换浅色 / 深色主题
- 切换数据周期
- 食品热量排序
- 展开图表数据表
- 新建运营规则弹窗及表单反馈
- 移动端侧栏
- `Command/Ctrl + K` 聚焦全局搜索

## 对照评估建议

建议固定使用以下维度给两版分别打 1–5 分：

1. 首屏能否在 10 秒内定位异常与机会。
2. 信息密度与阅读负担是否平衡。
3. 指标、趋势、目标和行动之间是否形成闭环。
4. 导航、筛选、审核等后台任务是否高效。
5. 视觉语言是否符合健康产品且不过度消费化。
6. 响应式、键盘操作、对比度与数据替代表达是否完整。

## 范围说明

本轮实现的是高保真总览页与后台全局框架。侧栏其他模块保留信息架构入口，不包含完整业务页面，避免在需求尚未定义时虚构大量业务流程。
