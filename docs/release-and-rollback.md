# 发布候选与回滚手册

M10 完成后仅代表进入发布候选阶段。生产发布仍须经过体验版灰度、真机验收和监控观察。

## 发布前门禁

1. `master` CI 全绿，`m10-complete` 指向已合并提交。
2. 生产环境通过 Secret 管理注入数据库、微信、JWT、删除审计 pepper、域名和 CORS 配置；仓库不保存真实值。
3. `SPRING_PROFILES_ACTIVE=prod`，确认 `/api/setup/seed` 返回 404，`/actuator/health` 返回 `UP`，限流已开启。
4. 备份 MySQL 8，并记录备份文件、Flyway 当前版本和校验和。
5. 在同版本数据库副本演练 V8 升级，核对 `user_goal` 行数与 `users` 行数一致，旧目标数据已迁移。
6. 微信体验版完成 iOS、Android 常见屏幕的登录、记录、编辑、删除、趋势、日历、目标与账号删除验收。

## 发布顺序

1. 进入维护窗口并创建数据库备份。
2. 部署后端，Flyway 自动执行 V8；先只开放内部健康检查流量。
3. 验证健康、登录、目标读写、首页聚合和匿名删除审计。
4. 发布小程序体验版，按 5% → 25% → 100% 灰度观察错误率、P95 延迟、401/429 比例和数据库连接池。
5. 观察至少 30 分钟无异常后完成全量；保存发布提交、配置版本和验收记录。

## 回滚触发条件

- 健康检查连续失败 3 次。
- 5 分钟窗口 API 5xx 超过 2%，或 P95 超过 1 秒且持续 10 分钟。
- 用户隔离、营养计算、目标迁移或账号删除出现数据完整性问题。
- 登录、饮食记录等 P0 主链路不可用。

## 应用回滚

1. 立即停止灰度并把流量切回上一稳定后端镜像。
2. 小程序撤回审核中版本或切回上一体验/线上版本。
3. 保留 requestId、匿名事件 ID、时间和错误分类；不得把 Token、openid、昵称或业务正文写入工单。

## 数据库回滚

V8 是增量迁移，旧 `users` 目标列仍保留一个版本用于紧急回退。若 V8 之后尚未产生新的目标修改，可执行 U8；脚本会删除两张 V8 表并移除对应的 Flyway 历史记录，使旧版应用能够按 V1–V7 校验并启动：

```bash
mysql --host="$DB_HOST" --user="$DB_USERNAME" --password "$DB_NAME" \
  < backend/src/main/resources/db/rollback/U8__user_goals_and_deletion_audit.sql
```

若已经产生新目标数据，禁止直接运行 U8。先停止写入，把 `user_goal` 导出并回填旧列或恢复发布前备份，再由负责人批准恢复服务。账号删除不可逆，回滚不会恢复已经确认删除的数据。

## 监控与告警

- 可用性：`/actuator/health`，1 分钟一次。
- API：按路由和状态码统计请求量、5xx、401、429 与 P50/P95。
- 数据库：连接池、慢查询、Flyway 状态、磁盘和备份。
- 文件：上传失败率、头像目录容量与孤儿文件巡检。
- 隐私：账号删除只记录匿名用户散列、事件 ID、requestId、结果和时间，默认保留 180 天并定期清理。
