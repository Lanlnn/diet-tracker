## 真机矩阵风险接受

本机 USB、Xcode 设备列表和开发者工具均未发现实体 iOS/Android 或现成远程调试会话，因此未把模拟器结果记作真机。随后已生成指向可访问 API 的预览二维码供用户扫码；用户反馈“真机实测还是不太行”，并于 2026-07-15 明确要求继续下一步目标。

因此 M10 不把真机矩阵标记为通过，而是按用户决策记录为 `waived` 的已知风险并继续 PR、合并和标签流程。后续补测仍应覆盖 iOS、Android 各至少两种常见屏幕的场景 A–F；若发现 P0/P1，立即停止发布并记录缺陷。
