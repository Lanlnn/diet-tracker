## 阶段状态

状态只允许使用：`未开始`、`进行中`、`待验收`、`已完成`。

当前所有阶段默认是`未开始`。只有代码、迁移、测试、文档和验收全部完成并推送远端后，才能改成`已完成`。

## Git 交付约定

每个阶段使用独立分支：

```text
codex/m0-foundation
codex/m1-design-system
...
codex/m10-profile-release
```

阶段交付流程：

1. 从最新主分支创建阶段分支。
2. 只提交本阶段范围内的代码、迁移、测试和文档。
3. 本地构建和验收清单通过。
4. 使用约定式提交，例如 `feat(m4): implement calorie calculator`。
5. 推送阶段分支：`git push -u origin <branch>`。
6. 合并后打标签：`m0-complete` 至 `m10-complete`。

禁止在没有实现和验证的情况下创建“完成”提交或标签。

## 每阶段完成定义

- UI 与指定优化版设计图一致。
- 正常、加载、空、错误、禁用状态齐全。
- 接口 DTO、鉴权、校验和用户隔离完成。
- 数据结构变化包含可回滚迁移。
- 自动化测试和阶段冒烟测试通过。
- API、架构和阶段文档同步。
- 不包含真实密钥、敏感日志和临时调试入口。
- 阶段分支已推送到 Git 远端。
