`07-餐次详情.png`

重点元素：餐次总热量、营养摘要、食品明细、编辑入口、继续添加和规则化建议。

## 目标

完成饮食记录创建、查询、编辑、删除和餐次聚合，形成第一条完整业务闭环。

## 数据与后端

- `MealRecord` 保存输入数量、单位、餐次、备注和营养快照。
- 保存时校验食品属于系统库或当前用户。
- 增加记录编辑接口和用户隔离查询。
- 删除返回明确的 204；不存在与越权分别返回 404/403。
- 写接口支持幂等键或客户端请求 ID。
- 餐次聚合只读取快照字段。

## 小程序

- 保存成功后进入对应餐次详情或返回首页。
- 展示餐次总热量和三大营养素。
- 支持继续添加、修改数量、修改餐次和删除。
- 删除二次确认，成功后立即刷新。
- 建议只根据当前餐次数据生成，数据不足时不显示。

## 接口

- `POST /records`
- `GET /records?date=&mealType=`
- `PUT /records/{id}`
- `DELETE /records/{id}`

## 测试与验收

- [x] 创建、编辑、删除后聚合数据一致。
